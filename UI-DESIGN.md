# SQL 平台 UI 设计文档

> 版本: v3 | 日期: 2026-05-02 | 状态: 评审修订 v4

---

## 1. 设计原则

- **效率优先**：80% 用户 80% 时间在查询页，查询体验是第一优先级
- **不打断工作流**：AI 评审内联展示，不弹窗不阻塞（除非高风险）
- **开发友好**：深色主题、语法高亮、快捷键、代码补全
- **渐进披露**：默认简洁，需要时展开详情

## 2. 整体布局

左侧导航 + 右侧工作区，经典 SaaS 三栏布局。

```
┌───────────────────────────────────────────────────────┐
│ 🗄️ SQLFlow   [🔍 搜索]  [📦 生产]  [🔔 待审批 3]  [👤▼] │
├────────┬──────────────────────────────────────────────┤
│ 📊 查询 │  页面标题 / 面包屑                           │
│ 📋 工单 │                                              │
│ 🔒 权限 │           工作区内容                          │
│ 📜 审计 │                                              │
│ ⚙️ 设置 │                                              │
│        │                                              │
│  [◀]   │                                              │
└────────┴──────────────────────────────────────────────┘
  ↑折叠                                            展开→
```

### 顶部导航栏

- **左侧**：Logo + 产品名 "SQLFlow"
- **中间**：全局搜索入口（Cmd+K 快捷键，搜索工单/审计日志/SQL）
- **右侧（左到右）**：环境指示器（生产，红色 badge）→ 通知铃铛（待审批工单数量角标）→ 用户头像 + 下拉菜单（个人设置 / 修改密码 / 登出）
- **主题切换**：用户下拉菜单内，MVP 阶段仅深色主题可用

### 左侧导航

- **展开态**（默认，~200px）：图标 + 文字，显示完整菜单名
- **折叠态**（~56px）：仅图标，hover 时 tooltip 显示菜单名
- **折叠按钮**：侧边栏底部，点击切换展开/折叠
- **导航项**：
  - 📊 查询（默认首页，路由 `/query`）
  - 📋 变更工单（路由 `/tickets`，待审批数量角标）
  - 🔒 权限管理（路由 `/permissions`）
  - 📜 审计日志（路由 `/audit`）
  - ⚙️ 设置（路由 `/settings`，子菜单展开：数据源 / 脱敏规则 / AI 配置）
- **活跃态**：当前页面对应菜单项高亮（`--accent-primary` 左侧边框 + 背景色）
- **分割线**：设置与上面导航项之间用 `--border-subtle` 分隔

```
│   │   └──────────────────────┘   │   │
│   │                              │   │

│   │                              │   │
│   │    © 2026 SQLFlow           │   │

│                                      │
└──────────────────────────────────────┘
```

**视觉设计：**
- 背景 `--bg-base`，可选微妙网格图案或低对比度渐变装饰
- 登录卡片 `--bg-surface` + `--shadow-lg` + `--radius-xl`
- Logo 区域与表单之间间距 `--space-8`（32px）

**交互说明：**
- 输入框支持 Enter 键快速登录
- 登录失败：输入框下方显示红色 inline 错误提示「用户名或密码错误」（不使用 toast）
- 登录中：按钮显示 spinner + 文字变为「登录中...」，按钮 disabled 防止重复提交
- 登录成功：JWT 存储在 localStorage，跳转到查询页
- Token 过期：任意 API 返回 401 → 统一拦截 → 清除 localStorage → 跳转登录页，显示 toast「登录已过期，请重新登录」
- MVP 不做「记住我」功能

**表单验证：**
- 用户名：必填，长度 3-32 字符
- 密码：必填，长度 8-128 字符，至少包含字母和数字
- 实时校验：输入框失焦时校验格式
- 提交校验：点击登录时校验必填

## 3. SQL 查询页（核心页面）

### 页面结构

查询页采用**上下分割布局**，编辑器与结果表格通过可拖拽分割线分隔：

```
┌─────────────────────────────────────────────────────┐
│ 顶栏                                               │
├──────┬──────────────────────────────────────────────┤
│      │ [MySQL ▾] [业务库 ▾]        [+ Tab] [🕐 历史] │
│      │ ┌─ Tab1 ─┬─ Tab2 ─┬─ Tab3 ─┐                  │
│      │ │                            │                  │
│ 侧   │ │  CodeMirror SQL 编辑器     │                  │
│  边   │ │  (flex: 可拖拽调整)        │                  │
│  栏   │ │                            │                  │
│      │ ├────────────────────────────┤                  │
│      │ │ 🤖 AI 评审卡片（条件显示）   │                  │
│      │ ├────────────────────────────┤                  │
│      │ │ ⏱️ 0.23s | 87 行 | 影响 0  │ [▶ 执行] [📤] │
│      │ ├─────── ═══════ 拖拽分割 ═══ │                  │
│      │ │                            │                  │
│      │ │  结果表格                   │                  │
│      │ │  (flex: 可拖拽调整)        │                  │
│      │ │                            │                  │
│      │ ├────────────────────────────┤                  │
│      │ │ [← 1 2 3 ... →]  共 87 行  │                  │
│      │ └────────────────────────────┘                  │
└──────┴──────────────────────────────────────────────┘
```

### 布局规则

- **分割线可拖拽**：默认 50:50，用户可自由调整编辑器与结果表格的高度比
- **无结果时**：结果区域显示空状态提示（图标 + 「执行查询以查看结果」），分割线和结果区域始终可见
- **首次查询后**：自动切换为 50:50
- **记忆偏好**：分割比例存储在 localStorage，刷新后保持
- **最小高度**：编辑器和结果表格各自最小 120px

### 多 Tab 查询

- Tab 栏位置：数据源选择栏下方、编辑器上方
- 每个独立保存：数据源选择 + 编辑器内容 + 查询结果 + 分割比例
- Tab 操作：可关闭（× 按钮）、可新增（+ 按钮），最后一个不可关闭
- 未保存标记：编辑器内容有修改但未执行的 Tab，标题显示 `•` 圆点
- Tab 标题：自动截取 SQL 前 20 字符，hover 显示完整 SQL

### 查询历史

- **入口**：Tab 栏右侧时钟图标 🕐 按钮
- **形式**：点击展开 Dropdown 面板（max-height 400px，可滚动）
- **内容**：最近 200 条查询记录（后端存储，跨设备同步）
  - SQL 摘要（前 60 字符）
  - 执行时间
  - 数据源名称
  - 执行耗时
- **存储**：后端 SQLite（按用户隔离，超出 200 条自动清理最旧记录）
- **操作**：
  - 点击某条 → 在新 Tab 中打开该 SQL + 自动选择对应数据源
  - 右键/更多按钮 → 删除单条记录
  - 面板底部 → 「清空历史」按钮（二次确认）

### 3.1 数据源选择区

- 数据库类型下拉（MySQL / MongoDB），切换类型后编辑器自动切换语法模式
- 数据库名下拉（带搜索），选择后自动加载该库的表名列表（用于自动补全）
- 环境标签（生产，红色高亮）

### 3.1.1 MySQL 查询

- 编辑器为标准 SQL 模式（CodeMirror 6 + MySQL 语法高亮）
- 用户直接输入 SQL 语句
- 支持 SELECT / DDL / DML，非 SELECT 操作提交时提示走工单
- 表名、字段名自动补全（基于所选库的元数据）

### 3.1.2 MongoDB 查询

- 编辑器切换为 JSON 模式（CodeMirror 6 + JSON 语法高亮）
- 提供三个输入区域：
  1. **操作类型**下拉：find / aggregation / update（走工单）
  2. **Filter / Pipeline** JSON 编辑器：查询条件或聚合管道
  3. **Options** JSON 编辑器（可折叠）：projection、sort、limit 等选项
- 预设常用模板（空 filter 查询全部、按 ID 查询等）
- find 操作直接执行，update 操作提示提交工单
- 集合名通过下拉选择（基于所选数据源的库列表）

### 3.2 SQL 编辑器

- **组件**：CodeMirror 6
- **功能**：
  - 语法高亮（MySQL / MongoDB 查询语法）
  - 自动补全：表名、字段名、关键字
  - 格式化：一键美化 SQL
  - 多 Tab：支持同时打开多个查询
  - 历史查询：保存最近 N 条，可快速恢复

### 3.3 AI 评审结果区

用户点击「执行」后触发 AI 评审，结果以内联卡片形式展示在编辑器与执行状态栏之间。评审期间编辑器保持可用（可继续编辑 SQL），执行按钮显示加载状态。

- 🟢 **低风险** → 评审完成后 1s 自动执行，绿色卡片展示结果
- 🟡 **中风险** → 评审完成后黄色卡片提示风险，用户确认后执行
- 🔴 **高风险** → 评审完成后红色卡片，执行按钮置灰，卡片底部显示「提交工单」主按钮

卡片内容包括：
- 评审状态（加载中 / 完成）
- 风险等级（带颜色标识）
- 优化建议列表（可折叠，流式追加）
- 影响分析（预估扫描行数、变更行数）
- 操作按钮（根据风险等级不同）：
  - 低风险：自动执行，显示「已完成」+ 查看结果
  - 中风险：「确认执行」+ 「修改 SQL」+ 「提交工单」
  - 高风险：「提交工单」（主按钮，accent 色）+ 「返回修改」

**高风险提交工单流程：** 点击评审卡片的「提交工单」→ 弹出抽屉填变更原因 → 提交（不跳转页面，保持用户在查询上下文）

### 3.4 执行状态栏

- 执行耗时
- 返回行数
- 影响行数（DML）
- 脱敏提示（如有敏感字段被脱敏）

### 3.5 结果表格

- 列宽可拖拽
- 列排序（点击表头）
- 列搜索/筛选
- 分页（默认 50 行/页，可调整：50 / 100 / 200）
- 空结果友好提示（图标 + 「未查询到数据」文字）
- 大结果集警告（超过配置阈值时提示）
- 脱敏字段：表头显示脱敏图标 🔒，hover tooltip 说明「该字段已脱敏」
- 单元格内容过长：截断 + tooltip 显示完整内容

### 3.6 快捷键

**MySQL 模式：**

| 快捷键 | 功能 |
|--------|------|
| Ctrl + Enter | 执行 SQL |
| Ctrl + S | 保存查询（暂存） |
| Ctrl + Shift + F | 格式化 SQL |
| Ctrl + Space | 触发补全（表名/字段名/关键字） |
| Ctrl + E | 导出结果 |

**MongoDB 模式：**

| 快捷键 | 功能 |
|--------|------|
| Ctrl + Enter | 执行查询 |
| Ctrl + S | 保存查询（暂存） |
| Ctrl + Shift + F | 格式化 JSON |
| Ctrl + Space | 触发补全（集合名/字段名） |
| Ctrl + E | 导出结果 |

### 3.7 导出交互

- **入口**：执行状态栏的「📤 导出」按钮
- **格式选择**：点击展开 Dropdown → CSV / JSON 两种格式
- **前置检查**：
  - 无查询结果时按钮 disabled
  - 超过 10,000 行时弹确认框：「当前结果共 N 行，导出可能耗时较长，是否继续？」
- **导出过程**：选择格式后直接触发文件下载（后端流式返回），不阻塞页面
- **大数据量**：导出过程中状态栏显示「正在导出...」，完成后 toast 提示「导出完成」

## 4. AI 评审交互设计

### 评审流程

> 用户点击「执行」后才开始 AI 评审，**不在输入时实时触发**。
> 评审结果通过 SSE（Server-Sent Events）流式返回，前端逐步展示。

```
用户输入 SQL → 点击执行
  │
  ├─ 本地静态规则检查（毫秒级）
  │   ├─ 拦截（DROP DATABASE 等）→ 直接报错
  │   └─ 通过 → 继续 AI 评审
  │
  ▼
SSE 连接建立，AI 评审流式返回（3-10s）
  ├─ 🟢 低风险 → 评审完成后自动执行
  ├─ 🟡 中风险 → 评审完成后提示风险，用户确认后执行
  └─ 🔴 高风险 → 评审完成后强制走工单，执行按钮置灰
```

### 风险判定依据

- **低风险**：简单 SELECT，有 LIMIT，非敏感表
- **中风险**：复杂 JOIN、大表全量查询、无 LIMIT、查敏感表但无脱敏豁免（结果已脱敏）
- **高风险**：涉及敏感表 + 有脱敏豁免（查看原始数据）、疑似拖库模式、DDL/DML

### 评审卡片样式（SSE 流式）

**评审中（SSE 接收中）：**
```
┌──────────────────────────────────────────┐
│ 🤖 AI 评审中...              ⏳ 流式接收中 │
│ ◠ ◡ ◠ （脉冲动画，不确定进度）             │
│                                          │
│ 💡 已收到建议（实时追加）:                  │
│ • 建议给 created_at 加索引...            │
│ • ...                                    │
│                            已收到 3 条    │
└──────────────────────────────────────────┘
```

**评审完成：**
```
┌──────────────────────────────────────────┐
│ 🤖 AI 评审                    耗时: 2.1s │
│                                          │
│ 风险等级: 🟢 低风险（免审自动执行）       │
│                                          │
│ 💡 优化建议:                              │
│ • 建议给 created_at 加索引，当前全表扫描   │
│ • LIMIT 已设置，符合规范                  │
│                                          │
│ 📊 预估: 0 行变更 | 扫描 ~12,000 行       │
│                                          │
│        [▶ 执行]  [✏️ 修改]  [📋 提交工单]  │
└──────────────────────────────────────────┘
```

## 5. 变更工单页

### 列表页

```
┌────────────────────────────────────────────────────┐
│  变更工单                    [+ 提交新工单]        │
├────────────────────────────────────────────────────┤
│  [全部] [待审批(3)] [已通过] [已拒绝] [已取消] [已执行] │

│  筛选: [数据源 ▾] [提交人 ▾] [AI风险 ▾] [时间范围]  │
│  快捷: [我提交的] [待我审批]                       │
├─────┬──────────┬──────┬───────┬──────┬─────────────┤
│ #ID │ SQL 摘要 │ 库/表 │ AI风险 │ 状态 │ 提交时间    │
│ W001│ ALTER ...│ order │ 🟡 中 │ 待审 │ 04-22 10:30 │
│ W002│ UPDATE ..│ users │ 🔴 高 │ 已通 │ 04-21 16:20 │
│ W003│ CREATE ..│ log   │ 🟢 低 │ 已执 │ 04-21 14:00 │
└─────┴──────────┴──────┴───────┴──────┴─────────────┘
```

- **状态 Tab**：切换 + 未读计数角标
- **快捷筛选**：「我提交的」「待我审批」— 角色视角的常用过滤
- **高级筛选**：数据源、提交人、AI 风险等级、时间范围
- **搜索**：按 SQL 内容模糊搜索（输入框 debounced，300ms 延迟）
- **SQL 摘要**：截取前 50 字符 + 末尾省略号，hover 显示完整 SQL（tooltip max-width 400px）
- **行 hover**：浅色背景高亮 + 鼠标变为 pointer

- **评论**：纯文本输入（MVP 不做 markdown），支持 Enter 发送，Shift+Enter 换行
- **审批通知**：审批操作通过后自动推送到钉钉 webhook

### 新建工单

两个入口：
1. 工单列表页顶部「+ 提交新工单」按钮 → 打开独立页面 `/tickets/new`
2. 查询页 AI 评审高风险卡片内「提交工单」→ 弹出抽屉（SQL 自动填入，保持在查询上下文）

新建工单表单字段：
- 数据源选择（必填，下拉）
- SQL 编辑器（必填，复用 CodeMirror 6）

## 11. 全局交互规范

### 统一错误处理

- **401 Unauthorized**：统一拦截，清除 localStorage JWT，跳转登录页，toast 提示「登录已过期，请重新登录」
- **403 Forbidden**：显示「您没有访问此页面的权限」+ 返回按钮
- **404 Not Found**：显示 404 插画 + 返回首页按钮
- **网络断开**：页面顶部显示全局 banner（红色，`--z-sticky`），文案「网络连接已断开，部分功能不可用」
- **5xx Server Error**：toast 提示「服务器错误，请稍后重试」，记录 console.error
- **请求超时**：toast 提示「请求超时，请重试」

### 二次确认机制

以下操作需二次确认弹窗（Shadcn/ui AlertDialog）：
- 删除数据源
- 删除用户
- 驳回工单（必须填写原因）
- 导出超过 10,000 行
- 重置密码



### Cmd+K 全局搜索

- 快捷键 Cmd+K / Ctrl+K 打开全局搜索弹窗（Command Palette 风格）
- 搜索范围：工单（按 SQL 内容）、审计日志（按 SQL/表名）、快速跳转到页面
- 搜索结果高亮匹配文字，Enter 跳转
- **实现优先级：P2**（MVP 阶段快捷键注册但搜索弹窗可简化为仅页面跳转，后续接入后端搜索 API）
