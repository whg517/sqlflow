## v2.0 设计原则

1. **双主题一致性**：浅色/深色主题共享同一套语义 token，切换时零闪烁，组件无差异
2. **渐进增强**：所有新交互在基础功能之上叠加，降级优雅（如补全失败不阻塞编辑）
3. **键盘优先**：核心操作（搜索、补全、筛选）均有完整键盘支持
4. **信息密度不变**：管理平台信息密度优先，新增功能不挤占已有内容空间
5. **动画克制**：仅用于状态反馈（主题切换、弹出/收起），不做装饰性动画

## 新增/变更交互设计

### 1. 主题切换交互

#### 1.1 切换入口设计

**入口位置**：顶栏头像下拉菜单，新增「主题」菜单项，位于「修改密码」之上。

```
┌──────────────────────────┐
│  👤 zhangsan              │
│ ──────────────────────── │
│  ☀️ 浅色主题            ▸ │   ← 点击切换
│  🔑 修改密码              │
│  🚪 退出登录              │
└──────────────────────────┘
```

**交互逻辑**：
- 当前为深色主题时，菜单项显示 `☀️ 浅色主题`（Sun icon），点击切换至浅色
- 当前为浅色主题时，菜单项显示 `🌙 深色主题`（Moon icon），点击切换至深色
- 图标使用 Lucide `Sun` / `Moon`（size 14）
- 点击后下拉菜单自动关闭

#### 1.2 切换动画

- **无过渡动画**：主题切换通过 CSS 变量瞬间生效（`< 16ms`），不使用渐变/淡入动画
- **原因**：管理平台追求响应速度，且 CSS 变量切换是纯渲染层操作，天然无闪烁
- **CodeMirror 编辑器**：因使用 `EditorView.theme()` 读取 CSS 变量，切换时编辑器自动跟随，无需额外动画

#### 1.3 各组件适配规范

| 组件 | 适配方式 | 注意事项 |
|------|---------|---------|
| 布局（Layout） | CSS 变量自动生效 | `bg-[var(--bg-sidebar)]`、`border-[var(--border-default)]` 等 |
| 侧边栏 | CSS 变量自动生效 | `--bg-sidebar`、`--border-subtle` |
| 按钮（Button） | Shadcn/ui token 已映射 | `--color-primary`、`--color-destructive` 等 |
| 表格（Table） | CSS 变量自动生效 | `--bg-surface`、`--border-default` |
| 弹窗（Dialog/Sheet） | Shadcn/ui token 已映射 | `--color-popover` |
| CodeMirror 编辑器 | `sqlTheme` / `jsonTheme` 读取 CSS 变量，无需修改 | `--bg-surface`、`--text-primary`、`--accent-primary` |
| Select/Input | Shadcn/ui token 已映射 | 背景色、文字色、边框色 |
| Toast（Sonner） | Shadcn/ui theme 集成 | 自动跟随 |
| 审计行展开 | CSS 变量自动生效 | `--bg-elevated`、`--border-default` |

**浅色主题特殊处理**：
- 代码块（`<pre>`）浅色模式下背景使用 `--bg-elevated`（gray-100），而非纯白，保持视觉层次
- AI 评审卡片浅色模式下阴影使用 `--shadow-md`（浅色版本：`0 4px 6px rgba(0,0,0,0.07)`）

### 2. SQL 编辑器自动补全交互

#### 2.1 补全触发方式

| 触发方式 | 条件 | 说明 |
|---------|------|------|
| **自动触发** | 输入 ≥ 2 个字符 | CodeMirror `autocompletion` 扩展，`activateOnTyping` 延迟 300ms |
| **手动触发** | `Ctrl+Space` / `Cmd+Space` | 任何光标位置均可触发 |
| **点号触发** | 输入 `.` 字符后 | 用于触发表名.字段名补全（如 `users.`） |

**取消补全**：`Esc` 键关闭弹出框，继续输入无关字符自动关闭。

#### 2.2 补全弹出框设计

补全弹出框使用 CodeMirror 内置 `@codemirror/autocomplete` 的 tooltip 样式，通过自定义主题适配 SQLFlow 视觉语言。

```
┌─────────────────────────────────────────────┐
│  SELECT * FROM us█                           │
│              ┌──────────────────────────┐    │
│              │ 📋 users              T  │    │  ← T = table 标签
│              │ 📋 user_logs           T  │    │
│              │ 📋 user_sessions       T  │    │
│              │ ─────────────────────────│    │  ← 分隔线
│              │ 🔑 USING              K  │    │  ← K = keyword 标签
│              └──────────────────────────┘    │
│                                              │
└─────────────────────────────────────────────┘
```

**点号触发（字段补全）**：

```
┌─────────────────────────────────────────────┐
│  SELECT users.█                              │
│              ┌──────────────────────────┐    │
│              │ 🔢 id                 C  │    │  ← C = column 标签
│              │ 🔢 name               C  │    │
│              │ 🔢 email              C  │    │
│              │ 🔢 phone              C  │    │

│              │ 🔢 status             C  │    │
```

**补全弹出框样式**：

#### 2.3 键盘交互

| 按键 | 行为 |
|------|------|
| `↑` / `↓` | 在补全列表中上下移动选中项 |
| `Enter` | 确认选中项，插入补全文本 |
| `Tab` | 确认选中项，插入补全文本（同 Enter） |
| `Esc` | 关闭补全弹出框 |
| 继续输入 | 实时过滤补全列表 |

#### 2.4 类型标签/图标

补全项右侧显示类型标签，用于区分补全来源：

| 类型 | 标签文字 | 图标 | 颜色 |
|------|---------|------|------|
| 表名 | `T` | 无图标，标签样式 | `--risk-exec` (blue) 背景 |
| 字段名 | `C` | 无图标，标签样式 | `--accent-secondary` (violet) 背景 |
| SQL 关键字 | `K` | 无图标，标签样式 | `--text-muted` 背景 |
| MongoDB 操作符 | `O` | 无图标，标签样式 | `--risk-low` (green) 背景 |

**标签样式**：

#### 2.5 MongoDB 编辑器补全

MongoDB 编辑器（`MongoEditor.tsx`）的 JSON 模式补全：

| 补全类型 | 内容 | 触发条件 |
|---------|------|---------|
| 操作符 | `$match`, `$group`, `$project`, `$sort`, `$limit`, `$skip`, `$count`, `$unwind` | 输入 `$` 字符 |
| 集合名 | 从当前数据源获取的集合列表 | 在 `collection` 输入框（非 CodeMirror，使用标准 Select 组件） |

MongoDB 补全弹出框样式与 MySQL 一致，操作符类型标签为 `O`（绿色）。

## 交互动画规范

### 动画总表

| 交互 | 属性 | 时长 | 缓动函数 | 触发条件 |
|------|------|------|---------|---------|
| 主题切换 | 无动画（CSS 变量瞬间切换） | — | — | 点击主题切换项 |
| 补全弹出框显示 | opacity + translateY | 150ms | `--ease-out` | 自动/手动触发补全 |
| 补全弹出框隐藏 | opacity | 100ms | `--ease-in` | Esc / 失焦 / 选择 |
| 全局搜索弹窗打开 | opacity + scale(0.96→1) | 200ms | `--ease-out` | Cmd+K |
| 全局搜索弹窗关闭 | opacity + scale(1→0.96) | 150ms | `--ease-in` | Esc / 选择 |
| 列筛选面板打开 | opacity + translateY(4px→0) | 150ms | `--ease-out` | 点击筛选图标 |
| 列筛选面板关闭 | opacity | 100ms | `--ease-in` | 点击外部 / Esc |
| 审计行展开 | max-height + opacity | 200ms | `--ease-default` | 点击行 |
| 审计行收起 | max-height + opacity | 150ms | `--ease-default` | 再次点击行 |
| 侧边栏折叠/展开 | width | 200ms | `--ease-default` | 点击折叠按钮 |
| 链接 hover | color | 150ms | `--ease-default` | 鼠标 hover |

### 缓动函数参考

```
--ease-default: cubic-bezier(0.4, 0, 0.2, 1)   /* 通用 */
--ease-in:      cubic-bezier(0.4, 0, 1, 1)      /* 进入 */
--ease-out:     cubic-bezier(0, 0, 0.2, 1)      /* 退出 */
```

### 动画实现原则

- **所有动画均尊重 `prefers-reduced-motion`**：当用户系统偏好「减少动画」时，所有过渡时长降为 0ms
- **不使用 `setTimeout` 驱动动画**：统一使用 CSS transition 或 CSS animation
- **避免同时触发超过 3 个动画属性**：保持性能
- **补全/筛选弹出框使用 `transform` 和 `opacity`**：GPU 加速，不影响布局

```css
/* reduced-motion 适配 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    transition-duration: 0ms !important;
    animation-duration: 0ms !important;
  }
}
```

## §8 间距系统修正（2026-05-24）

> 基于 UI 视觉审查发现的间距层级问题，明确修正方案。

### 8.1 问题概述

当前实现存在以下间距问题：

1. **Card 内部 padding 偏小**：筛选栏 `py-2.5`（10px）、Tab 栏 `pt-3`（12px）太紧
2. **页面标题到内容区间距不足**：大量使用 `mb-4`（16px），应至少 `mb-5`（20px）
3. **组件层级间距无递进**：筛选栏到表格只有 `border-b` 分隔，没有额外间距
4. **双栏布局间距不足**：Settings 两栏之间 `ml-4`（16px）太紧
5. **`h-[calc(100%-48px)]` hack**：硬编码高度扣减导致内容被压缩

### 8.2 修正方案

#### 8.2.1 Layout 全局

| 元素 | 当前 | 修正为 | 说明 |
|------|------|--------|------|
| Layout main | `p-6` | `p-6`（保持） | 24px，L0 层级，合理 |
| 页面标题 | 各页面自定义 | 统一 `text-xl font-semibold mb-5` | L0→L1 间距 20px |

#### 8.2.2 Ticket（工单页）

| 元素 | 当前 | 修正为 |
|------|------|--------|
| 外层容器 | `h-[calc(100%-48px)]` | `h-full` |

| Card 容器 | `rounded-lg border bg-surface flex flex-col` | 保持，但内部间距调整 |
| Tab 栏 | `px-4 pt-3` | `px-5 pt-4` |
| 筛选栏 | `px-4 py-2.5` | `px-5 py-3` |
| 分页栏 | `px-6 py-2` | `px-5 py-3` |
