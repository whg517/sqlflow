# Proposal: 004-frontend-completion — 前端功能补全

> 状态: 已批准
> 创建日期: 2026-05-08
> 更新日期: 2026-05-08
> 关联 Spec: spec/PRD.md, spec/UI-DESIGN.md

## 概述

补全前端页面中缺失的功能模块，使 MVP 达到完整可交付状态。当前后端 API 已全部就绪，但前端仍有三个核心模块未实现：Admin 用户管理、Dashboard 概览页、查询→工单联动引导。

## 背景

Phase 0-3 已完成后端全部核心 API（认证、数据源、查询、工单、AI 评审、脱敏、权限、审计、通知）和大部分前端页面。经代码审查发现以下前端缺口：

1. **Admin 用户管理页面缺失** — 后端已有完整 CRUD API（`/api/users`），但前端无对应管理界面，admin 无法在 Web UI 上管理用户
2. **无 Dashboard 首页** — 用户登录后直接跳转到查询页，缺少全局概览（待审批工单数、近期查询统计、系统状态等）
3. **查询→工单联动 UX 断裂** — AI 评审判定高风险（DecisionTicket）时，前端仅展示结果，未提供一键创建工单的引导入口

## 范围

### 包含
- Admin 用户管理前端页面（CRUD + 重置密码）
- Dashboard 概览页（统计卡片 + 待办事项）
- 查询页 AI 评审高风险→工单创建联动
- Layout 侧边栏导航更新

### 不包含
- 后端 API 变更（全部已实现）
- 移动端适配
- 国际化

## Spec 变更

详见 `spec-delta.md`。

## 风险与依赖

- **低风险**：纯前端工作，不涉及后端逻辑变更
- **依赖**：后端 API 稳定（已实现并测试通过）

## 验收标准

1. ✅ Admin 可在 Web UI 完成 用户创建/编辑/禁用/重置密码 全流程
2. ✅ Dashboard 页展示关键统计（待审批工单、近 7 天查询数、活跃数据源等）
3. ✅ 查询页 AI 评审高风险时，提供「创建工单」按钮，自动填充 SQL/数据源等信息
4. ✅ 所有新页面遵循现有 UI 规范（暗色主题、Shadcn/ui 组件）
5. ✅ 新页面通过 TypeScript 类型检查，无 ESLint 错误
