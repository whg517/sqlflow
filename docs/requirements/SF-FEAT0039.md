## 需求概述

增加工单批量审批功能，允许审批人同时选择多条工单进行批量通过或批量拒绝。

## 评审结论

- **架构评审（方远）**：✅ 通过
- **UI/UX 评审（苏晴）**：✅ 通过

## 技术方案

### 现有功能

| 功能 | 实现 |
|------|------|
| 工单状态流转 | SUBMITTED → AI_REVIEWED → PENDING_APPROVAL → APPROVED/REJECTED |
| Dashboard | 展示 pending_tickets 计数 |
| 工单列表页 | 支持单条审批操作 |
| 权限校验 | 只有授权审批人可以操作 |

### 实施方案

#### 后端

| 改动 | 说明 |
|------|------|
| 新增 API | `POST /api/tickets/batch-approve`、`POST /api/tickets/batch-reject` |
| 参数 | `{ ticket_ids: number[], reason?: string }` |
| 权限校验 | 复用现有审批权限逻辑（中间件级别） |
| 审计日志 | 每条工单记录独立审批记录（遍历 ticket_ids） |
| 错误处理 | 部分成功部分失败返回详细结果 |

#### 前端

| 改动 | 说明 |
|------|------|
| 多选模式 | 工单列表页增加 checkbox 列 |
| 批量操作栏 | 全选/取消全选、批量通过、批量拒绝按钮 |
| 确认弹窗 | 批量操作前二次确认（显示选中数量） |
| 审批意见 | 可选输入框 |

### 工时估算

| 任务 | 工时 |
|------|------|
| 后端批量 API + 权限 + 审计 | 3h |
| 前端多选 + 批量操作栏 + 弹窗 | 2h |
| 测试验证 | 1h |
| **合计** | **6h** |

## 验收标准

1. 工单列表页支持 checkbox 多选
2. 批量操作栏显示：全选、批量通过、批量拒绝
3. `POST /api/tickets/batch-approve` 正确更新选中工单状态
4. `POST /api/tickets/batch-reject` 正确更新选中工单状态
5. 批量操作记录独立审计日志
6. 权限校验：无审批权限用户无法操作
7. 部分失败时返回详细错误信息
