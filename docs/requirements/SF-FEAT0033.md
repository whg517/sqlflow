## 技术方案

### 接口设计

| 接口 | 说明 |
|------|------|
| GET /api/analytics/overview | 总览统计（查询量/工单数/活跃用户趋势） |
| GET /api/analytics/users | 用户维度排行（查询次数、工单数、敏感操作） |
| GET /api/analytics/datasources | 数据源维度统计（使用频率、慢查询分布） |
| GET /api/analytics/trend | 查询量趋势图数据（日/周/月粒度） |
| GET /api/analytics/export | 导出报表 CSV |

### 前端

- 新增 Analytics 页面（路由 /analytics）
- 复用现有 ECharts 依赖（Dashboard 已引入）
- 图表类型：
  - 折线图：查询量趋势
  - 柱状图：用户排行、数据源使用频率
  - 饼图：操作类型分布
- 时间筛选组件：快捷选择（今日/本周/本月/自定义范围）

### 查询优化

- 审计日志表已有 created_at 索引
- 趋势查询按日聚合，减少数据量
- 复杂聚合查询限制时间范围（最多 90 天）

## 验收标准

1. 查询趋势图 + 用户排行 + 数据源统计展示正确
2. 时间范围筛选功能完整
3. 报表导出 CSV 正常
4. 大数据量（审计日志 10w+ 条）响应时间 < 3s
