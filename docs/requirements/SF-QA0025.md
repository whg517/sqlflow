## 安全要求（韩锐）

1. **路径脱敏**：文件路径展示时脱敏敏感目录信息
2. **RBAC 权限控制**：覆盖度数据按角色控制访问（QA/开发/技术负责人不同权限）
3. **上传鉴权 + 解析器防御**：LCOV 文件上传需鉴权，解析器需防御恶意格式
4. **审计日志**：覆盖率数据变更操作需记录审计日志

## 架构实施建议（方远）

1. **API 版本化**：`/v1/` 前缀，为后续扩展预留空间
2. **色阶可配置**：阈值和对应颜色支持配置化，不硬编码
3. **解析器容错**：LCOV 解析器需容错处理（格式异常不崩溃，记录错误跳过）
4. **CI 并行 job 合并**：多个 CI job 产出的 LCOV 报告需支持合并后再入库
5. **安全要求落地**：上述 4 项安全要求纳入技术方案设计

## 验收标准

### V1 验收（12h）

1. 能解析 Go 和 JS 的 LCOV 格式覆盖率报告
2. 项目 → 模块 → 文件三级聚合数据正确
3. 基础展示页面正确展示模块级覆盖度，色阶标注准确
4. 覆盖度低于阈值的模块有醒目标注
5. 数据持久化到 PostgreSQL，历史快照可查询
6. 原始数据保留 90 天，聚合趋势数据永久保留

### V2 验收（14h）

1. E2E 覆盖度可通过 Playwright + 前端插桩方案采集
2. 仪表盘正确展示项目/模块/文件三级覆盖度数据（含 E2E 维度）
3. 趋势图支持日/周/月粒度切换，数据准确
4. 覆盖度门禁可在 CI 流水线中生效（warn/block 模式）
5. 报告可导出为 Markdown 格式
6. 页面视觉与 SQLFlow 设计系统一致
