# SQLFlow 需求文档

本目录按需求编号存放每个需求的详细文档，是项目的事实源（Single Source of Truth）。

## 文档生命周期

1. **评审通过时创建** — 评审结论、技术方案、设计规范落文档
2. **开发前确认** — 开发 Agent 必须确认对应需求文档已就绪
3. **上线后补充** — 实际实现记录、偏离说明、已知问题

## 文件命名

`{需求编号}.md`，例如 `SF-FEAT0031.md`

## 文档模板

每个需求文档应包含以下章节（按需裁剪）：

```markdown
# {需求编号} {需求名称}

> 状态：{待文档/待开发/开发中/已上线}
> 优先级：{P0/P1/P2/P3}
> 负责人：{Agent 姓名}
> 创建日期：{YYYY-MM-DD}
> 上线日期：{YYYY-MM-DD}

## 需求概述

{一句话描述}

## 评审结论

{评审通过/有条件通过/需修改，评审意见摘要}

## 技术方案

{架构设计、技术选型、接口设计、数据库变更}

## 设计规范

{UI/UX 设计要点（前端需求）}

## 验收标准

{可验证的完成条件}

## 实现记录

{上线后补充：实际实现、偏离说明、已知问题}

## 变更记录

| 日期 | 变更内容 |
|------|----------|
```
