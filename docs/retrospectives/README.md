## 规范

### 命名

`RETRO-NNNN-<简短英文描述>.md`

### 模板

每篇复盘包含以下章节：

1. **时间线** — 从隐患埋入到问题暴露到最终修复的完整链路
2. **技术根因** — 深入分析为什么会发生，引用相关规范/文档
3. **为什么反复踩坑** — 心理因素、流程漏洞、知识盲区
4. **本应在哪一步拦住** — 逐层分析防线为什么失效
5. **关键教训** — 提炼为可执行的认知更新
6. **行动项** — 具体的改进措施，带负责人和优先级
7. **参考** — 相关链接和 commit

### 原则

- **对事不对人**：复盘目标是改进系统，不是追责个人
- **量化损失**：用具体数字（时间、commit 数、文件数）衡量影响
- **可执行行动**：每篇复盘必须产出至少 1 个行动项
- **持续跟踪**：行动项的完成状态需定期回顾
